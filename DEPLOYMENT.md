# 🚀 2026 문화공연 사냥꾼 배포 및 공유 가이드 (Deployment Guide)

본 프로젝트는 경량화 백엔드(Starlette, Uvicorn, SQLite)와 정교한 바닐라 프론트엔드로 기획되어 배포가 매우 용이하도록 설계되었습니다. 조별 활동 발표 및 공유를 위한 **1) 초간단 로컬 네트워크 공유 링크**와 **2) 무료 클라우드(Render.com)를 통한 영구 웹 배포** 방법을 친절히 안내합니다.

---

## 1. 🛜 발표용 초간단 로컬 네트워크 공유 링크 (LAN Sharing)

같은 장소(클래스룸, 동아리방, 미팅룸 등)에서 같은 와이파이(Wi-Fi)나 공유기 허브에 접속되어 있는 조원들은 별도 가입이나 복잡한 설정 없이 **호스트 PC의 IP 주소**를 통해 실시간으로 시스템에 접속할 수 있습니다!

* **현재 가동 중인 발표용 공유 링크**:
  ```text
  http://100.73.31.116:8000
  ```
* **사용 방법**:
  1. 호스트 PC에서 `python server.py`가 켜져 있는지 확인합니다 (현재 켜져 있습니다!).
  2. 같은 공유기 Wi-Fi를 사용하는 조원들의 스마트폰, 노트북, 아이패드 브라우저에 위 주소(`http://100.73.31.116:8000`)를 입력하면 즉시 접속됩니다.

---

## 2. ☁️ 무료 클라우드 배포 (Render.com Docker 일성 배포)

클라우드 플랫폼 서비스인 **Render.com**은 Dockerfile을 자동으로 해석하여 무료로 전 세계에서 언제나 접속 가능한 공인 HTTPS 링크를 생성해 줍니다. (완전 무료이며 가입에 1분밖에 걸리지 않습니다!)

### 🛠️ 3단계 배포 절차
1. **GitHub 저장소 생성 및 푸시**:
   * 본 프로젝트 폴더의 내용(`Dockerfile`, `requirements.txt`, `server.py`, `static/` 폴더 등)을 본인의 깃허브(GitHub) 신규 레포지토리에 업로드(Push)합니다.
2. **Render.com 연동 및 웹 서비스 생성**:
   * [Render.com](https://render.com)에 로그인 후, 우측 상단의 **`New +`** 버튼 ➡️ **`Web Service`**를 선택합니다.
   * 방금 생성한 깃허브 레포지토리를 연결해 줍니다.
3. **빌드 설정 지정 (자동 매핑)**:
   * **Runtime**: `Docker`로 자동 인식됩니다. (혹시 수동 선택 시 `Docker` 지정)
   * **Branch**: `main` 혹은 `master`
   * 하단의 **`Deploy Web Service`** 버튼을 클릭하면 배포가 자동으로 시작됩니다!

> **💡 배포 완료 후 결과**:
> 빌드가 약 2-3분 내에 성공하며, 화면 좌측 상단에 `https://culture-hunter-xxxx.onrender.com` 과 같이 전 세계 어디서든 접속 가능한 **고유한 무료 영구 배포 HTTPS 링크**가 발급됩니다! 이 링크를 조별 보고서 및 발표 피드백용으로 공유하시면 완벽합니다.

## 📂 배포 준비 파일 목록
* **`requirements.txt`**: 프로젝트 구동에 필수적인 가벼운 종속성 라이브러리 목록
* **`Dockerfile`**: 운영체제 환경에 구애받지 않고 컨테이너 내부에서 데이터베이스를 선배치(`python import_csv.py`)하고 웹 서버를 무결하게 자동 시동하는 인프라 정의 파일
